1. نظرة عامة على المشروع وأهدافه الهندسة

يمثل هذا المستند وثيقة تصميم البرمجيات (Software Design Document) لمنظومة الجامعة الموحدة (UIMP) متكاملة مع النظام الفرعي لإدارة المجموعات البحثية (RGMS). تم بناء وتصميم هذا النظام لتلبية المتطلبات الإدارية والأكاديمية والبحثية للجامعة بكفاءة عالية، مع الالتزام التام بالفصل المعماري الهيكلي للمنظومتين تلبيةً للتوجيهات الأكاديمية التي تفرض استقلالية كل نظام مع الحفاظ على الترابط التكاملي بينهما.

1.1 أهداف الفصل المعماري (Decoupling Objectives)

يهدف الفصل الهيكلي للمشروعين إلى تحقيق عدة مزايا هندسية رئيسية:

  • تقليل الاعتماد المتبادل (Loose Coupling): يضمن أن أي عطل أو صيانة في نظام البحوث لا يؤثر بأي شكل من الأشكال على استقرار نظام الجامعة الأساسي الذي يحتوي على بيانات الطلاب والرواتب.
  • تسهيل التطوير والصيانة (Maintainability): يمكن لفرق برمجية مختلفة العمل على تطوير النظامين بشكل متوازٍ، حيث يمتلك كل نظام مخزن كود (Repository) مستقل.
  • تحسين الأداء وتوزيع الأحمال (Scalability): يمكن استضافة كل نظام على خادم منفصل وبموارد مخصصة تناسب حجم الطلبات عليه.

2. البنية المعمارية وهيكلية التواصل (System Architecture)

يعتمد النظام في بنيته المعمارية على إطار عمل Laravel (الإصدار 11) متضمناً هيكلية محسنة تدمج بين البساطة والأمان العالي.

2.1 تقسيم الخدمات ومنافذ التشغيل

تم عزل النظام إلى تطبيقين مستقلين تماماً:

النظام منفذ التشغيل المحلي الوظائف الرئيسية الوصول الأمني
نظام الجامعة الأساسي (UIMP Core) http://127.0.0.1:8000 الطلاب، الموظفون، الهيكل التنظيمي، إدارة الصلاحيات، وسجلات التدقيق. مفتوح لجميع الحسابات حسب الصلاحية المحددة.
نظام البحوث الفرعي (RGMS Subsystem) http://127.0.0.1:8001 المجموعات البحثية، المشاريع، المنشورات، الميزانيات، وحجوزات المعامل. يقتصر على الأكاديميين والباحثين والإدارة فقط.

2.2 آلية تسجيل الدخول الموحد (Single Sign-On - SSO)

لتحقيق تجربة مستخدم سلسة تمنع مطالبة المستخدم بتسجيل الدخول في كل مرة ينتقل فيها بين المنفذين، تم تطبيق الميكانيكية التالية:

  1. تطابق مفتاح التشفير (Shared APP_KEY): تم ضبط كلا المشروعين لاستخدام نفس المفتاح المشفر في ملفات التكوين لضمان قدرة النظامين على تشفير وفك تشفير ملفات تعريف الارتباط (Cookies) الخاصة بالجلسات.
  2. مشاركة قاعدة بيانات الجلسات (Database Session Sharing): تم ضبط برنامج التشغيل للجلسات ليكون SESSION_DRIVER=database، مما يوجه النظامين لحفظ بيانات الجلسة النشطة في جدول sessions الموحد.
  3. تكامل المتصفح: نظراً لأن كلا النظامين يعملان على نفس النطاق المحلي (IP: 127.0.0.1)، يقوم المتصفح تلقائياً بإرسال ملف تعريف الارتباط الخاص بالجلسة لكلا المنفذين، مما يحقق عبوراً أمنياً فورياً وتلقائياً.

3. قاموس قاعدة البيانات الموحد (Data Dictionary)

يشترك النظامان في قاعدة بيانات علائقية واحدة تدعم الربط المباشر والسلامة المرجعية للبيانات.

3.1 جدول المستخدمين (users)

يخزن هذا الجدول الحسابات وبيانات الاعتماد الأساسية للمنظومة.

اسم الحقل نوع البيانات مفتاح الوصف
id UUID Primary Key المعرف الفريد للمستخدم (تم استخدام UUID لزيادة الأمان).
username VARCHAR(50) Unique اسم المستخدم لتسجيل الدخول.
email VARCHAR(100) Unique البريد الإلكتروني الرسمي للمستخدم.
password_hash VARCHAR(255) - كلمة المرور المشفرة بخوارزمية BCRYPT.
is_active BOOLEAN - حالة الحساب (نشط / معطل).
locked_until TIMESTAMP - وقت قفل الحساب في حال إدخال كلمة المرور خاطئة 5 مرات متتالية.

3.2 جدول سجلات الطلاب (students)

يحتوي على السجلات الأكاديمية وحالة قيد الطلاب بالجامعة.

اسم الحقل نوع البيانات مفتاح الوصف
id UUID Primary Key المعرف الفريد للطالب.
institutional_id VARCHAR(50) Unique رقم القيد الجامعي للطالب.
name_ar VARCHAR(150) - اسم الطالب باللغة العربية.
name_en VARCHAR(150) - اسم الطالب باللغة الإنجليزية.
enrollment_status VARCHAR(30) - حالة القيد (ACTIVE, SUSPENDED, GRADUATED, UNDER_REVIEW).

3.3 جدول المجموعات البحثية (research_groups)

يخزن البيانات الأساسية للمجموعات البحثية المسجلة بنظام البحوث.

اسم الحقل نوع البيانات مفتاح الوصف
id UUID Primary Key المعرف الفريد للمجموعة البحثية.
code VARCHAR(30) Unique الرمز المميز للمجموعة البحثية (مثل AI-RG-01).
name_ar VARCHAR(150) - اسم المجموعة باللغة العربية.
name_en VARCHAR(150) - اسم المجموعة باللغة الإنجليزية.
primary_discipline VARCHAR(100) - التخصص البحثي الرئيسي للمجموعة.

4. جدار الحماية والصلاحيات الأمنية (RBAC Control)

يعتمد النظام على سياسة تحديد الصلاحيات بناءً على الأدوار (Role-Based Access Control) لمنع تداخل العمليات الإدارية والبحثية.

4.1 مصفوفة الوصول للأدوار (Access Control Matrix)

توضح المصفوفة التالية مستويات الوصول المصرح بها لكل مستخدم في كلا النظامين:

الدور (Role) إدارة الطلاب (UIMP) إدارة الموظفين (UIMP) سجلات التدقيق (UIMP) إدارة البحوث (RGMS)
SYSTEM_ADMIN الوصول الكامل الوصول الكامل قراءة فقط الوصول الكامل
REGISTRAR الوصول الكامل محجوب محجوب محجوب بالكامل (ممنوع من الدخول)
HR_STAFF محجوب الوصول الكامل محجوب محجوب بالكامل (ممنوع من الدخول)
AUDITOR قراءة فقط قراءة فقط قراءة فقط قراءة فقط لمراجعة الميزانيات
FACULTY قراءة الملف الشخصي قراءة الملف الشخصي محجوب إضافة وإدارة مجموعاته البحثية الخاصة

5. حماية سجلات التدقيق ومنع التلاعب (Audit Immutability)

تعتبر سجلات التدقيق الأمني (Audit Logs) جوهر الرقابة والنزاهة في المنظومة، ولهذا تم تطبيق مفهوم **عدم القابلية للتغيير أو الحذف (Immutability)**.

5.1 حماية قاعدة البيانات عبر الـ Database Triggers

لمنع أي محاولة لتغيير أو مسح السجلات، تم حقن Trigger برمجي في محرك قاعدة البيانات يطلق استثناءً يمنع أي عملية تعديل أو حذف:

CREATE TRIGGER prevent_audit_log_updates
BEFORE UPDATE ON audit_logs
FOR EACH ROW
BEGIN
    SIGNAL SQLSTATE '45000' 
    SET MESSAGE_TEXT = 'سجلات التدقيق الأمني غير قابلة للتعديل نهائياً.';
END;

CREATE TRIGGER prevent_audit_log_deletes
BEFORE DELETE ON audit_logs
FOR EACH ROW
BEGIN
    SIGNAL SQLSTATE '45000' 
    SET MESSAGE_TEXT = 'سجلات التدقيق الأمني غير قابلة للحذف نهائياً.';
END;

6. مواصفات واجهات برمجة التطبيقات (API Specifications)

تتم التفاعلات والمشاركة الخارجية للبيانات من خلال واجهات RESTful APIs قياسية.

6.1 جلب المجموعات البحثية (Get Research Groups)

رابط استرجاع كافة المجموعات البحثية النشطة بصيغة JSON:

GET /api/v1/rgms/research-groups
Authorization: Bearer [Sanctum_Token]
Accept: application/json

نموذج الاستجابة المتوقعة (Response JSON):

{
    "success": true,

    ]
}

7. دليل التنصيب والتشغيل السريع (User Guide)

لتشغيل النظامين معاً على جهاز كمبيوتر جديد، اتبع الخطوات الهندسية التالية:

7.1 تشغيل النظامين بنقرة واحدة

قم بفتح المجلد الرئيسي للمشروع C:\UIMP-RGMS واضغط مرتين على الملف start-all.bat. سيقوم هذا الملف البرمجي بتشغيل الأوامر التالية تلقائياً:

@echo off
# تشغيل سيرفر نظام الجامعة الأساسي على منفذ 8000
start "UIMP Core" /D "uimp-core" php artisan serve --host=127.0.0.1 --port=8000

# تشغيل سيرفر نظام البحوث الفرعي على منفذ 8001
start "RGMS Subsystem" /D "rgms-subsystem" php artisan serve --host=127.0.0.1 --port=8001

اترك النوافذ السوداء مفتوحة لتستمر السيرفرات في معالجة طلبات المتصفح بنجاح.
